## 1.什么是中断  
轮询模式：当CPU正在执行主程序时，每经过一个固定的时间间隔(轮询间隔，可以是1秒，可以是5秒)，CPU就会不断去读取引脚，导致CPU效率极低，实时性很差。  
  
中断模式：相当于你增加了一个中断触发源，当中断产生时，CPU会保存当前执行的任务进度去执行中断，导致中断的任务执行完后再去执行刚刚做的事，中断让CPU从“主动查询”变成“被动响应”，只有在发生特定事件时才去处理，其余时间可以安心做自己的事，极大地解放了CPU。  
## 2.中断源：谁可以触发中断  
外部中断：触发者为引脚上的电平变化。比如：按键按下（GPIO下降沿触发）、传感器信号跳变。  
  
内部中断：触发者为芯片内部模块发出的信号。比如：串口（UART）收到数据（RXNE标志）、定时器溢出（Time out）、ADC转换完成、DMA传输完成。  
## 3.STM32的最小系统包含什么  
电源、复位、时钟、BOOT引脚  

电源：电源旁一般需要放两个电容，一个0.1μF的陶瓷电容，另一个为 10μF ~ 22μF 的电解电容。电容必须紧贴芯片引脚，小电容(主要负责滤波)紧贴芯片引脚，大电容(充当小水池，主要用于稳压)放稍远一点。  

复位：一个 10kΩ 电阻上拉到3.3V，再加上一个 0.1μF 电容下拉到地，再接一个按键到地。  

时钟：用来给CPU提供心跳。   
主晶振（HSE，高速外部时钟）：通常接 8MHz 无源晶振，搭配两个 20pF ~ 22pF 的负载电容。经过芯片内部PLL（锁相环）倍频，最高可以跑到72MHz（F1系列）或上百MHz（F4/H7系列）。  
备用晶振（LSE，低速外部时钟）：通常接 32.768KHz 的晶振。专门给内部的RTC（实时时钟）使用，走时间用的。  

BOOT引脚：STM32有 BOOT0 和 BOOT1 两个引脚，决定芯片上电后从哪个地址读取程序。  
|BOOT0|BOOT1|启动位置|应用场景|
|---|---|---|---|
|0（接地）|X（任意）|主Flash（用户程序区）|正常工作模式（绝大多数情况）|
|1（接3.3V）|0|系统存储器（BootLoader）|串口下载程序（ISP下载，不用调试器）|
|1（接3.3V）|1|内置SRAM|调试用（极少用）|
## 4.二极管三极管和MOS管  
||二极管|三极管|MOS管|
|---|---|---|---|
|引脚|2个：阳极(A)、阴极(K)|3个：基极(B)、集电极(C)、发射极(E)|3个：栅极(G)、漏极(D)、源极(S)|
|控制本质|不可控（单向导通）|电流控制电流（必须给I<sub>B</sub>）|电压控制电流（只需给V<sub>GS</sub>，几乎不耗电流）|
|输入阻抗|低|低（几百Ω）|极高（MΩ级，近乎绝缘）|
|导通等效|固定压降（0.7V）|固定压降（V<sub>CE</sub>≈0.2~0.3V）|可变电阻（R<sub>DS(on)</sub>，如10mΩ）|
|温度特性|负温度系数|负温度系数（发热→电流更大→热击穿）|正温度系数（发热→电阻变大→电流自降，适合并联）|
|开关速度|慢（恢复时间长）|慢（电荷存储效应）|极快（无存储效应，适合高频）|
|典型应用|整流、续流|小信号放大、低速便宜开关|DC-DC电源、电机驱动、高频开关|

## 6.画板子要注意什么  
1.晶振要“抱紧”MCU：8MHz或32.768kHz的晶振，必须紧挨着MCU的OSC_IN/OUT引脚，距离越短越好。晶振下方的表层尽量不要走其它信号线（覆铜接地最好），否则高频振荡会干扰别的信号。  

2.电容要“贴”着引脚：0.1μF（104）去耦电容，必须紧挨着MCU每个VDD引脚放置，走线要先经过电容再到MCU引脚。如果离得远，电容等于白放，电源纹波会很大。  

3.走线要短，电源线加粗，不要走直角和锐角。  
## 7.单片机如何实现延时  
#### 1.软件延时（空循环）  
让 CPU 跑无意义的循环来"耗时间"。  
|优点|缺点|
|---|---|
|不依赖任何外设，代码极简|阻塞 CPU，延时期间什么都干不了|
|精度尚可（粗略场景够用）|受编译优化、主频变化影响大|  
⚠️ 现代编译器会优化掉空循环，需加 volatile 或关闭优化。  
  
#### 2.SysTick 定时器延时 —— HAL 库的标准做法  
STM32 内置的 SysTick（系统滴答定时器），专门用于产生延时，HAL 库底层就是用它。  
|优点|缺点|
|---|---|
|由硬件定时器驱动，相对准确|仍然阻塞 CPU（HAL_Delay 是轮询等待）|
|不占用额外定时器资源|在中断里调用 HAL_Delay 会死锁|  

#### 3.定时器中断延时 —— 最常用，不阻塞 CPU  
配置一个通用定时器，让它周期性地进中断，在中断里做标记。  
| 优点                  | 缺点        |
| ------------------- | --------- |
| **完全不阻塞 CPU**，多任务并行 | 需要配置定时器外设 |
| 精度高，可做到 µs 级        | 代码结构稍复杂   |
| 适合复杂项目              |           |
  
#### 4.RTOS 延时 —— 多任务/复杂项目  
使用 FreeRTOS、RT-Thread 等实时操作系统  
| 优点           | 缺点                     |
| ------------ | ---------------------- |
| 任务自动切换，代码最清晰 | 需要引入 RTOS，占用 RAM/Flash |
| 适合复杂多任务系统    | 学习成本高                  |

#### 5.四种方式对比
| 方式                     |  阻塞？  |  精度 | 复杂度 | 适用场景           |
| ---------------------- | :---: | :-: | :-: | :------------- |
| **软件空循环**              |  ✅ 阻塞 |  低  |  极简 | 教学演示、简单点灯      |
| **SysTick/HAL\_Delay** |  ✅ 阻塞 |  中  |  简单 | 初始化阶段、简单延时     |
| **定时器中断**              | ❌ 不阻塞 |  高  |  中等 | **最推荐**，绝大多数项目 |
| **RTOS 延时**            | ❌ 不阻塞 |  高  |  较高 | 多任务、复杂系统       |
